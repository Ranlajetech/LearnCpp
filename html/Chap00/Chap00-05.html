0.5 -- 编译器 / 链接器 / 库 简介

继续围绕上一节的流程图展开讨论。

本节讨论第 4 - 7 步。

第 4 步：编译源代码

编译 C++ 程序需要编译器。C++ 编译器逐个处理程序的源文件，并完成两项重要任务：

首先，它会检查代码是否遵循 C++ 语言的规则。如果没有，就会抛出编译错误（和对应的代码行号）来帮你找到哪里需要修改。编译过程也会被打断，直到修复错误。

然后，它会把 C++ 源码翻译成机器语言文件：目标文件。目标文件一般命名为 name.o 或 name.obj，其中 name 和原 .cpp 文件的名字相同。

如果程序有 3 个 .cpp 文件，编译器就会生成 3 个目标文件：

C++ 编译器在很多不同的操作系统上都可用。我们一会再介绍安装编译器，现在先不用管它。

第 5 步：链接目标文件和库文件

编译器生成完目标文件后，链接器开始工作。链接器的工作有三个方面：

其一，它会把编译器生成的所有目标文件合并成为一个可执行的程序。

其二，链接器除了可以链接目标文件，还可以链接库文件。库文件是一组提前编译打包好的、可被其他程序复用的代码文件。

C++ 核心语言实际上很小很简明（你可以在本教程中学到很多）。然而，C++ 还有一组称作 C++ 标准库（简称标准库）的拓展库提供了很多额外的功能，可以在程序中使用。C++ 标准库中最常用的一个是 iostream 库，它提供了在屏幕上显示文本与从键盘上获取用户输入的功能。几乎每个 C++ 程序都以某种形式使用了标准库，所以经常会把标准库链接到你的程序上。绝大多数链接器都会在你使用标准库的任意一部分时自动链接，所以你无需考虑这一部分。

你也可以选择链接其他库。比如写播放声音的程序，一般你不会想要自己写代码实现从硬盘读取声音文件、检查文件是否可用的功能，也不想深入探究怎么把声音数据传递给操作系统或者硬件再通过扬声器播放出来——这是个大工程！你可以下载个已经完成了这部分工作的库来用。在附录中讨论了怎么链接这些库和怎么创建自己的库。

其三，链接器会检查保证所有的跨文件依赖关系都被正确解析。例如，你在一个 .cpp 文件中定义了一些东西，然后在其他 .cpp 文件中使用了它，链接器就会把它们联系到一起。如果链接器没办法把引用和定义联系起来，就会抛出链接错误并中止链接过程。

链接器完成目标文件和库文件的链接之后，你就得到可执行文件去运行了！

延申阅读

对于复杂的项目，一些开发项目使用了 makefile 文件，它描述了如何生成程序（例如哪些文件要编译链接或者以其他方式处理）。有很多书教你怎么编写维护 makefile，这是个相当强大的工具。但 makefile 不是 C++ 核心语言的一部分，你也不需要使用它来学习本教程，本教程不会继续讨论它。

第 6 和第 7 步：测试和调试

这是个有趣的部分（希望如此）！运行你的可执行文件来看看它是否能产生预期的输出。

如果程序运行了但没得到正确结果，就得做点调试工作找出问题所在。我们之后会更详细地讨论怎么测试和调试你的程序。
