0.2 -- 编程语言简介

现代计算机非常之快，而且还在变得更快。然而，计算机也有一些明显的限制：它只懂得有限的一组命令，还必须被清晰明确地指示要做什么。

计算机程序，也称为应用，是能让计算机执行并完成任务的一组指令。开发这个程序的过程称为编程。程序员开发程序一般是通过编写源代码（简称代码）来完成，而后者是输入到一个或多个文本文件的一系列指令。

组成计算机、能进行计算的物理器件称为硬件。计算机程序被载入到内存，硬件按顺序执行每一条指令的过程称为运行或执行程序。

机器语言

计算机的 CPU 读不懂 C++。CPU 所能直接理解的那一组有限的命令称为机器码，也称为机器语言或指令集。

机器语言指令大概长这样：10110000 01100001

计算机刚发明出来的时候，程序员必须直接用机器语言写程序，这项工作非常困难、需要花费大量时间。

如何组织这些指令超出了本教程的范畴，但有两点值得注意。第一，每条指令都由 0 和 1 的序列组成。每个单独的 0 或 1 称为二进制数字，简称比特。构成单条指令的比特数可以不同，比如某些 CPU 处理的指令总是 32 位，而某些其他 CPU（比如 x86 系列，很可能你现在用的电脑就是）的指令长度是可变的。

第二，每组二进制数字都被 CPU 解释为执行特定工作的指令，比如比较这两个数字、把这个数字放到那个内存位置。然而，不同的 CPU 有不同的指令集，给某款 CPU 写的指令无法让拥有不同指令集的 CPU 执行。这就意味着程序不可移植（不大规模重写就没法用）到不同类型的系统上，只能从头重写。

汇编语言

因为机器语言太难懂难记，所以汇编语言诞生了。在汇编语言中，每条指令都用缩写标记，而不再使用二进制数字，也可以使用名称和数字了。

这条汇编代码和上面的二进制机器码表示相同的指令：mov al, 061h

汇编语言比机器语言要好读好写很多。但 CPU 不能直接理解汇编语言，汇编程序必须要翻译成机器语言才能被机器执行。这一过程由汇编器完成。用汇编写的程序都非常快，所以直到今天，对执行速度要求很高的程序仍然会用汇编来写。

然而，汇编语言也有它的劣势。首先，汇编语言仍然需要大量的指令才能做一点很简单的任务。尽管单条指令可读性还行，但想要理解整个程序仍很困难，这就有点像通过读一个一个字母来理解整个句子。第二，汇编语言的可移植性也很差，用一个 CPU 的汇编语言写的程序，一般不能在不同指令集的 CPU 上运行，所以必须重写或者作大量更改。

高级语言

为解决可读性和可移植性的问题，人们开发了新的编程语言，比如 C、C++、Pascal，还有后来的 Java、Javascript、Perl 等等。这些语言称为高级语言，程序员不再需要考虑编写的程序要跑在什么样的机器上了。

这条 C / C++ 语言的代码表达和上面相同的含义：a = 97;

和汇编程序类似，用高级语言编写的程序也得先翻译成机器能懂的格式才能执行。主要有两种途径：编译和解释。

编译器是个程序，它读入源代码，产生独立的可执行程序。当代码编译成可执行文件后，运行是不需要编译器的。早期的编译器很简陋，只能生成比较慢的未经优化的程序。而现代编译器已经可以生成快速的、优化过的程序，甚至某些情况下比人工写的汇编还要好。

绝大多数语言可以被编译或解释，但像 C、C++ 和 Pascal 等传统语言是要编译的，像 Perl、Javascript 这样的类脚本语言是要解释执行的。而某些语言，比如 Java，混合了这两种方式。

高级语言有很多优秀特性。

首先，高级语言因为它们的命令更像日常的自然语言而更易读写；其次，执行相同的任务，高级语言比低级语言所需的指令更少，这就让程序易于理解。C++ 中 a = b * 2 + 5; 可以简单地在一行里完成，用汇编语言就需要五六条不同的指令。

关于可移植性，有两个主要的例外。

第一，很多操作系统，比如微软 Windows，提供了一些平台专属的功能。这让面向特定操作系统的程序更容易开发，但代价是代码可移植性变差。本教程会回避任何基于特定平台的代码。

第二，很多编译器都支持一些编译器扩展。你的代码如果用了这些扩展，就没法在不作修改的前提下用不支持同样扩展的编译器编译。带你安装完编译器之后，我们再继续讨论这个话题。

规则、最佳实践、警告

在本教程中，我们会用这三种方式高亮显示重点内容。

规则

规则是你必须遵守的指示，这是由语言规定的。不遵守规则会导致你的程序不能正常运行。

最佳实践

最佳实践是你应该做的事，因为这么做一般被认为是标准做法或是推荐做法。要么是这种做法大家都在用（不这么做很可能会和他人预期相悖），要么是比其他做法更有优势。

警告

警告是你不应该做的事，因为这一般会导致不可预料的后果。
