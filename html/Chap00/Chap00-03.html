0.3 -- C/C++ 简介

在 C++ 之前，先来说说 C 语言

C 语言于 1972 年由 Bell Telephone Laboratories（贝尔实验室）的 Dennis Ritchie 创造，最初是个系统编程语言（用来编写操作系统的语言）。Ritchie 最初的目的是开发一个简单的语言，要易于编译、容易访问内存、代码高效、自成一体（不依赖于其他程序）。作为高级语言，要给程序员很多控制权限，同时还要做到独立（不需要为了移植而重写）于平台（硬件、操作系统等）。

1973 年，因为 C 很高效很灵活，Ritchie 和 Ken Thompson 用 C 重写了 Unix 操作系统的绝大部分。不像汇编那样写出来的程序只能运行在特定的 CPU 上，C 有完美的可移植性，使得 Unix 可以很简单地在不同种类的计算机上重新编译并加速其应用。C 和 Unix 的命运交织在一起，C 的流行在一定程度上和 Unix 的成功息息相关。

1978 年，Brian Kernighan 和 Dennis Ritchie 出版了 The C Programming Language（C 编程语言）一书，也称 K&R（作者姓氏的首字母）。此书提供了 C 语言的非正式规范，并成为了事实标准。当可移植性很重要时，程序员会严格遵守 K&R 里的推荐，因为当时绝大多数编译器实现都遵循 K&R 标准。

1983 年，美国国家标准协会 ANSI 成立了一个委员会来确立 C 的正式标准。1989 年，他们完成并发布了 C89 标准，也通常被称为 ANSI C。1990 年，国际标准化组织 ISO 接受了 ANSI C 并作了一小点修改，这一版又称 C90。编译器都遵从 ANSI C 或 C90 标准，因此最大可移植性的程序都使用这一标准。

1999 年，ANSI 发布了新标准 C99。这一版接受了很多编译器插件中的特性和 C++ 中一些已经实现的功能。

C++

从 1979 年开始，Bell Labs 的 Bjarne Stroustrup 开发了 C++ 作为 C 的拓展。C++ 向 C 添加了许多新特性，可以认为是 C 的超集，虽然并不准确（C99 添加了一些 C++ 中不存在的特性）。C++ 最开始出名是因为它是一门面向对象的语言。至于什么是“对象”，它们又与传统编程方式有什么区别，以后的章节里再说。

C++ 于 1998 年由 ISO 标准化。ISO 批准了一份 C++ 的描述文档，用来保证所有的编译器都遵循同样的标准。一个小更新于 2003 年发布（C++03 标准）。

到目前为止，C++ 有四次大更新：C++11，C++14，C++17，C++20，每次更新都加入了一些新功能，尤其是 C++11 标准加入了大量的新功能，被广泛认为是这门语言的一个新基点。大约每三年左右就有一次更新，C++23 新标准已经开始制定了。

这些正式发布的语言版本称为语言标准（或语言规范）。标准用发布年份命名，比如 C++15 不存在，因为 2015 年没有发布新标准。

C 与 C++

C 和 C++ 的潜在设计理念就是“信任程序员”。这很棒，但也很危险。C++ 的设计允许程序员做任何想做的事，同时也允许程序员做毫无道理的事，语言只会认为你是出于某种它不理解的原因故意这样做的。有很多陷阱，新程序员一不小心就容易掉进去。这就是“知道什么事不该做”和“知道应该怎么做”同样重要的原因。

Q：C++ 擅长用来做什么？

在需要高性能和精确控制内存和其他资源的情况下，C++非常出色。这里有一些比较可能要用 C++ 来写的情况：

  • 游戏
  • 实时系统（比如交通系统、生产控制等）
  • 高性能金融应用（比如高频交易）
  • 图形应用和模拟仿真
  • 生产力工具、办公软件
  • 嵌入式程序软件
  • 音视频处理
  • 人工智能、神经网络

Q：学这个教程之前我需要先学 C 吗？

不需要！你完全可以直接从 C++ 学起，你需要知道的这里都会教，包括那些需要避开的陷阱。

学会了 C++ 之后，再学标准 C 就很简单了。现在 C 主要用在一些小众场合，比如在嵌入式器件中的运行的代码、与只有 C 接口的其他语言协作等等。对于绝大多数其他情况，一般推荐使用 C++。
